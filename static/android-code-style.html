Grandstream 代码规范

普通代码规范:

google官方doc中给出非常详细的code style. 但是因为"外部的链接控制权在别人手上，今天是这个明天可能就是别的，或者干脆就没了。", 所以我把这个代码规范clone到了本地, 传送门.

补充:

涉及源码修改时的规则：

  • 代码规范，遵循源码原有的代码规范.
  • 修改代码， 需要在修改的代码前后添加以下注释.
        /*
              * @modify  by grandstream  
              * @oginal functions    将原先的代码注释 在这部分
              * @desc 说明修改的原因  
              * start modify
              */
 
             新的代码

             /*
              * end modify
              */
  • 加代码， 需要在修改的代码前后添加以下注释.
             /*
              * @add  by grandstream  
              * @desc set title for status bar
              * start add function
              */
 
             新的代码

             /*
              * end add function
              */
